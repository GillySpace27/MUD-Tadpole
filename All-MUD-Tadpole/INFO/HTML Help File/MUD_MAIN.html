MUD_MAIN

Your home base is 'MUD_MAIN.m'. Here's whats going on:


This main() function has four tasks.  

  • It acquires the reference trace: REF
  • It acquires the array of cross sections: MUDPIX
  • It analyzes each cross section and extracts its spectrum and spectral phase: MUDDAT
  • It intelligently concatenates all of the cross sections into a single pulse: PULSE


This function is used for both the experiment and the simulation, switched by the parameter 'Simulate'



PARAMETERS


  • Simulate (Logical)
    • This selects whether you want to use simulated data or live experimental data.
  • Plots (Logical)
    • Putting a 'true' here will cause tons of useful plots to run. They show the state of the data at several crucial points along the pathway. You may want to turn them on one at a time to see where each one comes from.
  • SaveMUDPIX (Logical)
    • A 'true' here will cause the program to write the images taken by the camera to disk.
  • MUDPIX_filename (String)
    • This will be the name of the file with the pictures.
  • MP_filetype (String)
    • Three possible inputs
    • 'bmp'
      • Saves each image as an individual bitmap file, with an index appended to the file name.
    • 'gif'
      • Saves all the images as a single animated gif.
    • 'both'
      • Does both.
